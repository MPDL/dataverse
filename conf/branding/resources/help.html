General

What is Edmond?

Edmond is the Open Research Data Repository for the Max Planck Society.

Where does the name "Edmond" come from?

The first collection in Edmond was a set of images of the nucleus of the comet Halley. Therefore, the repository was named after Edmond Halley (1656-1742), an English astronomer who is best known for computing the orbit of the eponymous Halley's Comet and after whom the comet is named.

What are the costs for using Edmond?

The usage of Edmond and also the Edmond support is free of charge to all members of the Max Planck Society. It is a service provided by the Max Planck Digital Library (MPDL).

How can I mention Edmond in my DMP?

You can mention Edmond as an open research data repository in your data management plan (DMP). Depending on the questions in your DMP, you can make use of the different answers given here. If you need more information or feedback on your DMP, please do not hesitate to contact our Edmond support team. They are happy to support you with everything you need.

Do you provide more information on research data management?

Yes, we do. Please visit our information platform on Research Data Management. If you have further questions regarding research data management, please contact our RDM support team.

Repository

Are published data in Edmond searchable globally?

Yes, Edmond is registered at the Registry of Research Data Repositories (re3data). Through re3data, Edmond is visible globally. Furthermore, DataCite is the DOI provider. So all metadata for all research data published on Edmond appear in the DataCite Commons. Hereby, all published datasets in Edmond are listed in a global search engine.

What other repositories do exist where I can publish my data?

The amount of available data repositories is increasing every day. Some repositories are discipline-specific, others are institutional like Edmond. The Registry of Research Data Repositories (re3data) collects all known data repositories. This list can also be sorted based on the different disciplines.

Is Edmond as repository certified?

Not yet, but Edmond is currently in a review process.

What metadata schema is used in Edmond?

The metadata schema in Edmond is oriented to the DataCite Metadata Schema 4.4. The metadata schema is generic. Special metadata needs might be mapped with the provided metadata fields and metadata fields for controlled vocabularies.

Account

Who can use Edmond?

All Max Planck employees and members can use Edmond. Likewise, all collaboration partners can use Edmond as well.

Is there a storage limitation for each account in Edmond?

No, there is no storage limitation in Edmond. However, it is advisable for larger amounts of data to use the DVUploader and no longer upload via the browser. Please contact our Edmond support team if you have any questions.

How is the login working?

Edmond uses the Single Sign-on system from the Administrative Headquarters of the Max Planck Society. That means all MPG accounts are ready to log in to Edmond. Additional registration is not necessary. External users can use the self-registration in Edmond.

How can I reset my password?

You can reset your password for Edmond. In the Single Sign-on (SSO) system, as an MPG employee and member, you can reset your password within your local IT infrastructure. Please contact your local IT staff for questions regarding your MPG account.

What roles and permissions are available in Edmond?

The following roles and permissions are available:
  • Admin
  • Contributor
  • Curator
  • File Downloader
  • Member
For a detailed description of the different roles and their permissions, please go to the following section in the generic Dataverse Guide.

Data Publishing

Does Edmond provide Digital Object Identifiers (DOI)?

Yes, Digital Object Identifiers (DOI) are automatically provided for each dataset version. The DOI is immediately reserved by creating a dataset. When a dataset is published, the DOI will then resolve. The same goes for each new version of an existing dataset.

Is it possible to reserve a Digital Object Identifier (DOI) for an unpublished dataset?

Yes, Edmond does this automatically. For each new dataset, a DOI will be reserved.

What are the benefits of depositing my research data?

  • Your data will be preserved, even for your own future use.
  • The publishing process creates a permanent record with a persistent identifier and a suggested citation, so that your work can be formally attributed when re-analysed by others.
  • Your data will be discoverable through search engines to maximize visibility and impact.
  • By providing access to your research data, you can fulfil the requirements of scientific journals, financial sponsors and other parties who have supported your research.

Why should I use Edmond for publishing my data?

Edmond is hosted by the Max Planck Digital Library, which means your data will not leave the Max Planck Society. The use of Edmond and the Edmond support is free of charge.

How can I share my dataset with others before publishing the dataset?

You can create a private URL for your unpublished dataset. With this link anyone can access your dataset without login. You can use this link i.e. to share it for a review process, when reviewers want to see your unpublished data in advance. For a detailed description of how to create a private link to your dataset, please see the following section in the generic Dataverse Guide.

Who can search and access a published dataset?

When you published your dataset, anyone can search and access it. Edmond is an Open Research Data Repository, which includes openness by design.

Can I withdraw my published dataset?

Yes, it is possible to withdraw a dataset after it was published. This can be done via the menu "Edit Dataset" and then the button "Deaccession Dataset". It is necessary to give a comment on why the dataset will be withdrawn. After that, only metadata and your comment are available for other users, but not your files anymore. The DOI is still resolving.

How can I cite a published dataset?

You can find a citation suggestion for the dataset at the top of the dataset page. You can also download the citation in different file formats.

Datasets

What information is mandatory for a dataset?

Four information are mandatory:
  • Title of the dataset
  • Author
  • Institution
  • Study Type

Can I use a folder structure for my dataset?

Yes, a folder structure is possible. You can generate the folder structure with different files in it. The DVUploader is also a comfortable way to create complex file structures.

Can I delete a dataset that I have created?

Yes, an unpublished dataset can be deleted. All information and files will be deleted and cannot be restored. A published dataset cannot be deleted, only withdrawn. Please see the description here.

Can I make changes to my published dataset?

Yes, changes on published datasets are always possible. You can i.e. add or delete files and also edit metadata. All will lead to a new version of the dataset, which needs to be published again. Depending on the changes, minor or major release versions of the dataset are available. The new version will receive a new DOI. Thereby it is fully available and citable.

Does Edmond provide versioning for datasets?

Yes, Edmond provides versioning of datasets. For published datasets you can select between minor and major versions, i.e. 1.1 or 2.0 and so on. For more information, please see the following section in the generic Dataverse Guide.

Under what licenses can I publish a dataset?

Edmond supports flexible licensing. Following licenses are recommended:
  • Creative Commons
    Creative Commons licenses were originally created for the legally secure licensing of creative content, not primarily for data. But since version 4, this slightly different nature of data is kept in mind so that CC-BY 4.0 and CC-BY- SA 4.0 are also usable for research data. The advantage of CC licenses is their wide distribution and awareness.
    By licensing your research outputs under CC-BY, your research is openly available, but it is required that others have to give you credit, in the form of a citation, should they use or refer to your research object.
    This license lets others distribute, remix, tweak, and build upon your work, even commercially, as long as they credit you for the original creation. This is the most accommodating of licenses offered. Recommended for maximum dissemination and use of licensed materials.
  • Open Data Commons
    Open Data Commons has developed standard licenses directly for data, therefore they are likely to fit a bit better. But they are just not as widespread as CC licenses.
  • Public Domain
    Not uncommon is the use of public domain waivers as CC0 or Open Data Commons Public Domain Dedication and License (PDDL) for research data. Here, the greatest possible freedom is granted in handling the data. Depending on the type of data, this can be useful, but it also leaves possibilities to misuse the data.
    Although CC0 does not legally require users of the data to cite the source, it does not take away the moral responsibility to give attribution, as is common in scientific research.
  • Datenlizenz Deutschland
    Through a collaboration between federal, state and municipal associations, Datenlizenz Deutschland has developed a recommendation for uniform conditions for the use of administrative data in Germany, which now exists as "data license Germany" in version 2.0. This version is officially accepted as an open license by the Council of Experts of the Open Definition.
A tabular overview of the different licenses can be found on the German page “Forschungslizenzen”.
If no license is manually assigned, the data within Edmond will automatically be released to the public domain under CC0.

Files

What types of data files can I upload?

You can upload all types of data files. There is no limitation. Of course, it should be in formats that make sense for the use and reuse of your dataset. Own software code can also be uploaded.

What are the size limits?

There are no size limits in Edmond. But please note that uploading large amounts of files via the browser can be time-consuming depending on the infrastructure. The DVUploader is specially designed to upload large amounts of files into Edmond. Please contact our Edmond support team if you have any questions.

Can I upload a zip file instead of many single files?

Yes, you can. By uploading a zip file, Edmond is detecting the corresponding folder structure and will represent this as folder structure in your dataset.

How can I delete a file?

Deleting an item from a published dataset will lead to a new version of the general dataset. See also the answer on versioning in Edmond.

How can I move a file?

You can move a file on the file level. This is possible via edit metadata, where also the "File Path" is available. Through this, moving a file to different folders is possible.

Technical

Which technology is used for Edmond?

Edmond uses the Dataverse software, an open-source community-driven development.

Is my data safe with Edmond?

Edmond uses diverse strategies to back up the data (back up of the server and of the database itself). These backups are created in frequent intervals (the shortest is every 12 hours) and are stored in several locations on servers of the Max Planck Society as well as on tapes.

Does Edmond provide a client?

Yes, you can use the DVUploader for that. For more information, please see the following section in the generic Dataverse Guide.

Does Edmond provide an API?

Yes, an API is available. The documentation of the Dataverse API is available in the generic Dataverse Guide.
